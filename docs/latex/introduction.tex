\section{Introduction}

\subsection{Architectural Directionality as an Inductive Bias}

Multilayer Perceptrons (MLPs) are traditionally structured as a unidirectional chain of connected layers, where each layer sequentially connects to the next \cite{cybenko1989approximation,hornik1989multilayer}.
This imposes a simple ordered sequence of transformations on the input data, where each layer defines a projection from one vector space to another, along with some activation function.
Hidden layers can then be interpreted as intermediate representations whose structure depends on the imposed ordering of the network.

Through these variations, many inductive biases have been successfully encoded such as convolutional locality, invariance to translation, rotation, scaling, and other geometric transformations \cite{lecun1998gradient,cohen2016group,worrall2017harmonic,jaderberg2015spatial,sabour2017dynamic}.
While many variations of this base model have been iterated on with respect to shared weights, activation functions, and other hyperparameters, the core architectural directionality of the network tends to remain the same and only a few deviations encode geometric transformations of the data, such as 2D or 3D convolution layers popularized in computer vision tasks.
They do still alter informational flow in substantially novel ways to produce these new inductive biases, but generally still have one input to output directionality.
Convolutional Neural Networks (CNNs) encode locality and translation equivariance through shared convolutional kernels \cite{lecun1998gradient,fukushima1980neocognitron}.
Extensions such as group-equivariant CNNs impose additional transformation structure, including rotation or reflection equivariance, by tying parameters across transformations \cite{cohen2016group,worrall2017harmonic}.

\subsection{Hexagonal Neural Networks}

Here, we introduce Hexagonal Neural Networks (HexNNs) that generalizes MLPs into a rotationally symmetric, multi-directional structure.

This architecture arranges nodes in a two-dimensional hexagonal topology rather than a single ordered layer stack, like the example shown in Figure \ref{fig:hexnet_n3_r0}.
This allows for the same set of nodes and edges to be interpreted and traversed under multiple directed orientations.
An orientation $H_i$ is a directed acyclic subgraph induced by choosing one of the six directed axes of the hexagonal topology.
Thusly, it is now possible to train one orientation while evaluating how that update affects the other orientations.
This yields a central question: whether this directional weight sharing produces measurable transfer or interference between orientations.

Because each orientation exposes input and output layers of equal width, the present formulation is most natural for tasks where input and output vectors have the same dimensionality, such as reconstruction, identity mappings, or vector-to-vector regression.
Classification tasks may still be represented, but may require additional post processing layers in case the number of features is not the same as the number of classes.

Given a size parameter, $n$, the network has equal $n$ length input and output layers, with hidden layers symmetrically increasing and decreasing to $2n-1$.
Crucially, each node in layer $i$ is connected to all nodes in layer $i+1$, but unlike MLPs, this bipartite adjacency is true for all three cardinal directions of a hexagon.
This allows for rotation-based reinterpretation of the same input data.

This design enables six distinct orientations of the same network, each corresponding to a $60^\circ$ rotation of the hexagonal structure.
We index orientations as $H_0, ..., H_5$ starting with the default orientation $H_0$ being the top-to-bottom orientation.
The opposite orientation of $H_i$ is $H_{(i+3) \bmod 6}$.

\subsection{Open Questions}

As we can see, this new change in architecture may have some interesting implications for the training of the network.
What are the implications of training the network in one direction, and how does that impact the training of the network in other directions?
What effects will this have on the performance of the network in other directions?
Most importantly, what new inductive biases, if any, can be encoded with this new architecture?

\subsection{Contributions}

In this paper, we present a memory efficient implementation using over padded state vectors and orientation specific adjacency matrices.
We also establish baselines for HexNNs and their equivalent MLPs and compare the performance of the two architectures across various datasets and hyper parameter configurations.
